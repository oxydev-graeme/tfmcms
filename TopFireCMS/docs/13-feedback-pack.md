# Feedback Pack

Use this pack to structure team feedback and capture clear decisions.

## Review agenda (60 minutes)

1. Context and goals (10 min)
2. MVP scope review (15 min)
3. Architecture and risk review (20 min)
4. Build vs buy discussion (10 min)
5. Decision and action items (5 min)

## Feedback prompts

Product:
- Is MVP scope realistic for 12 weeks?
- Which features are missing for pilot value?

Engineering:
- Is architecture appropriately simple for phase one?
- Are there major technical risks not captured?

Operations:
- Are reliability and incident plans sufficient?
- What observability gaps remain?

Security:
- Are permission and audit controls adequate for launch?
- What controls must be added before pilot?

## Scoring template

Score each area from 1 to 5:
- Strategic fit
- Technical feasibility
- Operational readiness
- Security confidence
- Time-to-value

## Decision outcomes

Select one:
- approve MVP build
- approve proof phase only
- defer and reassess later

## Suggested async process

1. Share the proposal docs in advance.
2. Ask reviewers to submit written comments before meeting.
3. Focus live meeting on unresolved questions and decisions.
4. Publish a decision memo within 24 hours.
