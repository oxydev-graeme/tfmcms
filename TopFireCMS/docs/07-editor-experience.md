# Editor Experience

## UX principles

- Clarity over complexity
- Guided workflows over hidden rules
- Fast actions for repeat tasks
- Confidence through previews and validation

## Core editor capabilities

1. Workspace-aware navigation
- Users only see assigned client workspaces.

2. Structured form editing
- Schema-driven forms with clear field labels.
- Inline validation with plain-language errors.

3. Draft and preview loop
- Save drafts automatically.
- Preview content before publish.

4. Review and approval flow
- Submit for review.
- Approvers accept or request changes.

5. Revision history
- Compare previous versions.
- Restore prior revision when needed.

## Workflow states

- draft
- in_review
- changes_requested
- approved
- scheduled
- published
- archived

## Collaboration model for MVP

- Assignment and comments on documents
- Task ownership per workflow step
- Notifications for pending approvals

## Accessibility and usability baseline

- Keyboard-friendly forms
- Strong contrast and readable typography
- Clear focus states and status chips
- Mobile-friendly review views

## Quality guardrails

- Required field enforcement
- Slug uniqueness checks
- Publish blockers for invalid references
- Workspace-level content conventions

## Editor metrics

Track and review:
- time from draft to publish
- rejection and rework rates
- validation error frequency
- workflow bottlenecks by step

## MVP user success criteria

- New editor can publish correctly within first day of onboarding.
- Approver can complete review process in under five minutes for standard content types.
- Content lead can identify stalled items in workflow at a glance.
