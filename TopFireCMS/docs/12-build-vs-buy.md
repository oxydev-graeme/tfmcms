# Build vs Buy Analysis

## Decision context

The agency must decide whether to:
- build TopFireCMS internally
- continue with existing third-party tools
- adopt a hybrid approach

## Option A: Build TopFireCMS

Advantages:
- Full control over roadmap and workflow fit
- Strong alignment with agency operating model
- Reduced long-term lock-in risk

Tradeoffs:
- Upfront engineering investment
- Ongoing platform ownership responsibility
- Need to establish internal support capability

## Option B: Buy and configure existing CMS stack

Advantages:
- Faster initial setup
- Lower short-term engineering effort
- Mature ecosystem from day one

Tradeoffs:
- Feature gaps for agency-specific workflows
- Recurring vendor costs and constraints
- Limited control over platform evolution

## Option C: Hybrid model

Approach:
- Use a proven CMS short term
- Build internal workflow and governance layers
- Gradually migrate core capabilities in-house

Advantages:
- Lower near-term delivery risk
- staged investment and learning curve

Tradeoffs:
- integration complexity
- dual-platform overhead during transition

## Recommendation

Given single-agency scope and desire for operational control, a focused internal build is viable if:
- MVP boundaries are enforced
- platform team ownership is clear
- pilot rollout is disciplined

## Decision criteria checklist

1. Can we staff a core team for at least 12 months?
2. Is editorial workflow differentiation strategically important?
3. Are we prepared to own reliability and security operations?
4. Will reusable client templates materially improve margin?

## Suggested next step

Run a four-week proof phase to validate:
- schema registry and migration model
- workspace isolation controls
- draft-to-publish workflow usability

Proceed to full MVP only if proof metrics pass agreed thresholds.
