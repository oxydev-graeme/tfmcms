# User Personas

## Primary personas

### 1) Agency Content Lead

Goals:
- Maintain editorial quality across clients.
- Ensure publishing policies are followed.
- Reduce rework and content errors.

Needs:
- Review queues and approval workflows.
- Clear validation feedback.
- Visibility into publishing status.

### 2) Agency Developer

Goals:
- Model content reliably.
- Ship frontend integrations quickly.
- Manage schema migrations safely.

Needs:
- Schema-as-code support.
- Type-safe API contracts.
- Migration and rollback tooling.

### 3) Client Editor

Goals:
- Create and update content without technical support.
- Understand what is required before publishing.

Needs:
- Simple forms with guided inputs.
- Draft and preview workflows.
- Permission-scoped navigation.

### 4) Client Approver

Goals:
- Validate accuracy and brand compliance before publish.

Needs:
- Approval queue.
- Change history and comparisons.
- One-click approve or request changes.

### 5) Agency Operations Owner

Goals:
- Keep platform healthy, secure, and cost-efficient.

Needs:
- Monitoring dashboards.
- Incident alerts.
- Backup and restore confidence.

## Persona priorities for MVP

Priority 1:
- Agency Content Lead
- Client Editor

Priority 2:
- Agency Developer
- Client Approver

Priority 3:
- Agency Operations Owner

## Key user journeys

1. Onboard new client workspace from blueprint.
2. Create schema changes with migration checks.
3. Draft, review, approve, and publish content.
4. Deliver content via API to frontend channels.
5. Audit user actions and recover from mistakes.
